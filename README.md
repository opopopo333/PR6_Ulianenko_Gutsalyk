# Тестирование банковского аккаунта (BankAccount Tests)

## Описание проекта
В данном проекте реализовано модульное тестирование класса `BankAccount` с использованием фреймворка в среде разработки **Microsoft Visual Studio**.

Тестирование проверяет корректность работы методов:
- `Debit()` — списание средств со счета
- `Credit()` — пополнение счета

Также проверяется корректность обработки исключений при неверных входных данных.

# Реализованные тесты

## 1. Debit_WithValidAmount_UpdatesBalance
Проверяет корректное списание средств со счета.

---

## 2. Debit_WhenAmountIsLessThanZero_ShouldThrowArgumentOutOfRange
Проверяет выброс исключения `ArgumentOutOfRangeException`, если сумма списания меньше нуля.

## 3. Debit_WhenAmountIsMoreThanBalance_ShouldThrowArgumentOutOfRange
Проверяет выброс исключения, если сумма списания превышает баланс счета.

---

## 4. Credit_WithValidAmount_UpdatesBalance
Проверяет корректное пополнение счета.

---

## 5. Credit_WhenAmountIsLessThanZero_ShouldThrowArgumentOutOfRange

## 6. Credit_WhenAmountIsLessThanZero_ShouldThrowArgumentOutOfRange_WithMessage

# Вывод о проведенном тестировании

В ходе тестирования были проверены основные сценарии работы банковского счета:

✔ корректное списание средств  
✔ корректное пополнение счета  
✔ обработка ошибок при отрицательных значениях  
✔ обработка ошибок при превышении баланса  

В процессе тестирования была обнаружена логическая ошибка в реализации методов обновления баланса.

В первоначальной версии программы:
- метод `Debit()` **увеличивал баланс**, хотя должен был его уменьшать;
- метод `Credit()` **уменьшал баланс**, хотя должен был увеличивать.

Из-за этого тесты, проверяющие корректность изменения баланса, выполнялись неуспешно.

После исправления кода:
- `Debit()` стал уменьшать баланс (`m_balance -= amount`);
- `Credit()` стал увеличивать баланс (`m_balance += amount`).

После внесения исправлений все тесты выполняются успешно.

---

# Причина успешного выполнения тестов

Тесты выполняются успешно, потому что:

- методы `Debit()` и `Credit()` корректно изменяют баланс счета;
- при передаче отрицательной суммы генерируется исключение `ArgumentOutOfRangeException`;
- при попытке списания суммы, превышающей баланс, также генерируется исключение;
- сообщения исключений соответствуют ожидаемым.

Это подтверждает правильность реализации логики банковского счета.
